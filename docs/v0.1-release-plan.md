# V0.1 Release Plan

V0.1 is the first usable product after MVP.

MVP proves the runtime. V0.1 makes it pleasant and credible for external users.

Status: `v0.1.0` was tagged and the repository checklist at the end of this
document records its local/CI evidence. The ZenMind/`agent-platform` bridge is
implemented and tested on branch `codex/zenforge-engine-bridge@d9ebc9e`, but
merge to platform `main`, deployment acceptance, and a real Container Hub smoke
test remain external items. Go 1.26.x is the only supported toolchain.

## V0.1 Goals

- stable-enough public SDK;
- polished CLI;
- documented safety defaults;
- durable local runs;
- useful examples;
- first ZenMind adapter spike;
- clearer provider support.

## ZenMind Adapter Spike

Goal:

Prove `agent-platform` can call ZenForge without replacing everything.

Approach:

1. Build `RunConfig` from ZenMind catalog/session through
   `zenmind.BuildRun`.
2. Project ZenForge events into the platform block lifecycle through a
   run-scoped `zenmind.Projector`.
3. Write event-only `root/chatId.jsonl` lines through
   `zenmind.NewChatJSONLWriter`; do not claim complete Chat Storage V3.1.
4. Use fail-closed AgentKey/ChatID/RunID routing through `zenmind.Router`.
5. Do not remove current runtime yet; `RouteLegacy` remains the default.

Acceptance:

- [x] Integration-branch evidence: `agent-platform` queries can run through the
  ZenForge engine selected by feature flags, with HTTP sync/async, SSE,
  WebSocket, approval, attach, and legacy-fallback tests at
  `codex/zenforge-engine-bridge@d9ebc9e`.
- [ ] Deployment acceptance: merge to platform `main` and verify the deployed
  UI receives recognizable ZenForge events.
- [x] Repository evidence: platform DTO, lifecycle, approval, and event-line
  goldens under `adapters/zenmind/testdata/platform` are pinned to
  `agent-platform@1893edb5` and exercised by `go test ./adapters/zenmind`.
- [x] Repository evidence: `adapters/zenmind.TestRouterFailsClosed` and
  `TestRouterRoutesOnlyExplicitInitializedZenForge` verify local fallback.

The adapter wire, JSONL event-line, approval, and routing contracts are covered
by ZenForge repository tests. The downstream bridge paths are covered by the
named platform branch tests. Platform-main deployment and real Container Hub
acceptance remain open and must not be inferred from either test suite.
