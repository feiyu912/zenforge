# V0.1 Release Plan

V0.1 is the first usable product after MVP.

MVP proves the runtime. V0.1 makes it pleasant and credible for external users.

## V0.1 Goals

- stable-enough public SDK;
- polished CLI;
- documented safety defaults;
- durable local runs;
- useful examples;
- first ZenMind adapter spike;
- clearer provider support.

## V0.1 Feature Set

Required:

- OpenAI-compatible adapter;
- JSONL checkpoint/event log;
- local workspace tools;
- shell tool;
- todo/planner;
- CLI approval;
- CLI config file;
- examples;
- documentation.

Strongly recommended:

- SQLite checkpoint/event store: `checkpoint/sqlite`, `eventlog/sqlite`;
- OpenTelemetry trace sink: `trace/otel`;
- basic sub-agent support: `subagent` and `tools/task`;
- Container Hub adapter beta: `sandbox/containerhub`;
- server/SSE helper package: `server/harnesshttp`, `server/sse`.

Optional:

- Anthropic adapter: `model/anthropic`;
- MCP adapter: `adapters/mcp`;
- memory adapter: `adapters/memory`.

## Public API Freeze

Before V0.1, review:

- root `zenforge.Config`;
- `Agent.Stream`;
- `Agent.Run`;
- `model.Model`;
- `tool.Tool`;
- `workspace.Workspace`;
- `checkpoint.Store`;
- `eventlog.Store`;
- `approval.Broker`.

Breaking changes are allowed before V0.1. After V0.1, breaking changes should be
batched and documented.

## ZenMind Adapter Spike

Goal:

Prove `agent-platform` can call ZenForge without replacing everything.

Approach:

1. Build `RunConfig` from ZenMind catalog/session.
2. Map ZenForge events to current stream events.
3. Keep chat JSONL as read model.
4. Use feature flag for selected agent.
5. Do not remove current runtime yet.

Acceptance:

- one internal agent can run through ZenForge path;
- UI receives recognizable events;
- fallback to old runtime remains available.

## Quality Bar

- tests pass on clean checkout;
- examples run locally;
- race-prone components tested with cancellation;
- docs mention unsafe configurations;
- CLI errors are actionable;
- no silent fallback from sandbox to host shell;
- no token leakage in default logs.
